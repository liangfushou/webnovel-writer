# Anti-AI 重写规则 v2.0（实测验证版）

## 核心理念

**AI写小说能不能赚钱，不取决于"AI会不会写"，而取决于：内容能不能让读者继续看、平台算法愿不愿意继续推。**

平台主要看：
- 1-5万字：点击率和前三章读完率
- 5-15万字：完读率
- 15-30万字：追更率和互动率
- 30万字以后：综合稳定性

**最关键指标：完读率**
- 70%以上：强推
- 50%以上：持续推
- 30%以下：基本凉

**AI文最大问题：不是"写得差"，而是"一眼AI"。**

---

## 实测验证的核心发现

**2026-04-27 实测数据：**
- 原版（自然混合段落）：59 疑似
- 版本4（长段落合并）：55.7 AI特征 ✅ **最优**
- 版本5（极简碎片）：72.7 AI特征 ❌
- 版本2（极度打散）：87.43 AI特征 ❌

**结论：适度合并相关句子，形成中长段落 = 降低AI特征。过度打散 = 增加AI特征。**

---

## 一级规则：段落与节奏（最关键）

### ✅ 正确的段落策略

1. **适度合并相关句子**
   - 描述同一场景/动作/情绪的句子合并成中长段落
   - 不要让每句话都独立成段（这是AI的典型特征）

2. **长短交替，不全短也不全长**
   - 句子要短，但不能全短
   - 环境铺垫可以用逗号连接
   - 情绪爆发再用句号停顿
   - 重要句子用句号单独停顿

3. **同一画面不要拆成碎片**
   - 一个完整动作/场景/情绪不要机械拆成多段
   - 保持叙事的自然连贯性

4. **连续同长度段落≥3：强制错落**
   - 避免连续3段以上长度完全相同
   - 段落长短应该由内容决定，不是为了"手机阅读"而机械切分

5. **超短句控制**
   - 每章超短句（≤5字）控制在5个以内
   - 单字台词（"嗯""好""哦"）每章≤3次

### ❌ 错误的段落策略（已证伪）

- ~~单段尽量不超过2行~~ ❌ 实测证明：长段落更低AI味
- ~~长段打散~~ ❌ 实测证明：过度打散反而增加AI味
- ~~关键压力点单独成段~~ ❌ 实测证明：不必强制独立
- ~~句长≤15字~~ ❌ 实测证明：会导致过度碎片化

## 二级规则：重复是大忌

### 单章重复阈值（严格控制）

| 类型 | 阈值 | 处理方式 |
|------|------|---------|
| 同一个动作描写 | 最多1次 | 第二次出现必须换说法 |
| 同一种情绪表达 | 不重复 | 换角度/换感官/换层次 |
| AI高频形容词 | ≤2次 | 替换为具体细节 |
| 模板化动作 | ≤2次 | 改为人设专属动作 |
| 空洞程度副词 | ≤3次 | 删除或改为精准动词 |
| 破折号 | ≤1处 | 多余的改为逗号或句号 |

### AI高频形容词黑名单

**外貌类：**
- 俊美、冷峻、深邃、修长、剑眉星目

**神态类：**
- 眸光一暗、薄唇轻启、瞳孔微缩、瞳孔一缩
- 死死盯着、倒吸冷气、眼底掠过、眸中闪过

**内心类：**
- 心中一凛、心中暗道、心头一跳、百感交集

**副词类：**
- 缓缓、淡淡、微微、轻轻、慢慢、渐渐
- 悄悄地、深刻地、根本地、显著地

**模板动作：**
- 咬嘴唇、指甲掐进掌心、嘴唇抿成一条直线
- 血液凝固、破风箱的呼吸、眼睛弯成月牙

### 跨章不能重复

- 不能复制粘贴开头、钩子、爽点结构
- 关键描写每章要换切入角度
- 语义高度相似的句子不要反复出现

## 三级规则：句式结构去模板化

### 绝对禁用（出现即删除）

| 句式 | 示例 | 处理方式 |
|------|------|---------|
| 否定式排比 | 不是…而是… / 不是X，不是Y，只是Z | 绝对禁用 |
| 自问自答 | X是什么？Y。 | 绝对禁用 |
| 指示型短语 | 首先/其次/最后、值得注意的是、此外、综上所述、关键是、总之、基本上 | 小说正文绝对禁用 |
| 充当式规避 | 作为…的象征、标志着、意味着 | 改为"是""有" |
| 模板桥段 | 壁咚、摸头杀、公主抱、拍卖会打脸、退婚、英雄救美 | 绝对禁用 |

### 打散同构句

若连续3句以上结构近似（主语+动词+宾语）：
- 至少切一处短句
- 或插动作锚点
- 或改成对白/动作/反问混排

### 删除段尾解释收束

以下句型优先删：
- 他终于明白了
- 她这才意识到
- 原来如此
- 这让他更加确定

### 截断情绪闭环

不要写成：起因 → 经过 → 结果 → 感悟

默认删掉最后的感悟句，保留压力与余味。

## 四级规则：对话驱动

### 对话占比

**网文不是文学小说，更像"对话驱动的故事"。**

- **对话占比建议：40%-60%**
- 但对话不能废，不能两个人寒暄五句不传递信息
- **每句对话都要推进剧情，最好带潜台词：角色说的和真正想的不一样**

### 对话必须具备（至少一项）

- 潜台词（说的和想的不一样）
- 打断
- 回避
- 反问
- 错位回答
- 沉默后的动作回应

### 对话禁止

- 对话完整复盘背景
- 全员书面语
- 严格 A-B-A-B 轮流且每句都正面回应
- 角色说话像在做报告

---

## 五级规则：环境与感官

### 环境描写要求

- 环境描写必须服务情绪、压迫感、推进或误导
- 无叙事功能的环境描写：补功能或删除

### 时空锚点

- 每章开头必有：时间/地点/天气 至少两项
- 避免时间流动感缺失

### 五感平衡

- 每章至少保留1处非视觉感官：冷、湿、痛、腥、灰、麻、刺等
- 避免视觉>80%

---

## 六级规则：黄金前三章

**前三章决定有没有第二轮推荐。**

### 第一章必须有

1. 主角当前困境或压力
2. 金手指线索（但不能拖太久）
3. 第一个冲突或悬念
4. 主角明确目标或欲望
5. 结尾钩子，让读者想看第二章

### 第一章绝对不要有

1. 大段背景介绍
2. 超过3个主要人物同时出现
3. 与主线无关的场景描写
4. 主角心理独白超过300字

### 黄金三章模型

**第1章：**
- 黄金3秒：开篇立刻抓人
- 前300字闭环：完成一个小情绪循环
- 500字内同框：主角与关键角色/冲突同框
- 1000字内金手指/情绪点：展示核心卖点
- 章末钩子：悬念/反转/压力

**第2章：**
- 开篇承接钩子
- 对话/行动立人设
- 第一个名场面与人物绑定
- 关系第一次递进
- 章末钩子

**第3章：**
- 开篇承接方式
- 冲突升级表现
- 第一个不可逆选择
- 回收伏笔
- 新埋伏笔≥2
- 章末钩子

---

## 七级规则：爽点密度

**读者耐心很短，不能让读者无聊。**

### 爽点分布

| 类型 | 密度 | 说明 |
|------|------|------|
| 打脸爽点 | 每10-15章 | 至少一次大高潮 |
| 逆袭爽点 | 每5-8章 | 金手指展示 |
| 甜宠爽点 | 每8-10章 | 感情线推进 |
| 身份曝光爽点 | 全文3-5次 | 不能太频繁 |

### 打脸标准结构

1. 建立压迫
2. 主角隐忍或准备反击
3. 打脸高潮
4. 围观者反应和主角金句收场

## 终检标准

以下任一不满足，`anti_ai_force_check` 应判 `fail`：

### 段落节奏
- [ ] 有中长段落，不是全部碎片化
- [ ] 长短句自然交替，不是全短或全长
- [ ] 同一画面未被拆成碎片
- [ ] 连续同长度段落<3段

### 重复控制
- [ ] 同一动作描写单章≤1次
- [ ] 同一情绪表达单章不重复
- [ ] AI高频形容词单章≤2次
- [ ] 模板化动作单章≤2次

### 句式结构
- [ ] 无否定式排比
- [ ] 无自问自答
- [ ] 无指示型短语
- [ ] 连续同构句已打散

### 对话质量
- [ ] 对话占比40%-60%
- [ ] 每句对话推进剧情或带潜台词
- [ ] 无全员书面语
- [ ] 无对话复盘背景

### 环境感官
- [ ] 每章有时空锚点（时间/地点/天气≥2项）
- [ ] 每章有非视觉感官描写≥1处
- [ ] 环境描写有叙事/情绪功能

### 章末处理
- [ ] 章末留未闭合压力或未解决问题
- [ ] 无安全着陆
- [ ] 无段尾感悟句

## 使用建议

1. **优先级：段落节奏 > 重复控制 > 句式结构**
2. **不要为了"去AI味"而过度修改，保持自然**
3. **同一画面不拆碎片是最重要的原则**
4. **长短交替比全部短句更像人写的**
