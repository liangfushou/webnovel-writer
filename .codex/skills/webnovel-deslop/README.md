# /webnovel-deslop 使用说明

## 快速开始

```bash
# 方式 1：润色文件
/webnovel-deslop 正文/第0001章-标题.md

# 方式 2：直接粘贴文本
/webnovel-deslop
[粘贴需要去AI味的文本]
```

## 功能说明

**用途**：单独对文本进行去AI味处理

**技术**：使用 DeepSeek 通用润色 + 番茄版排版（与 `/webnovel-chapter` 相同的两阶段润色）

**流程**：
1. 📊 AI味检测（生成检测报告）
2. 🎨 Stage 1: DeepSeek 通用去AI润色
3. 📱 Stage 2: 番茄版排版优化
4. 📄 输出润色结果

**预期时间**：1-2 分钟

## 两阶段润色详解

### Stage 1: DeepSeek 通用去AI润色

**做什么**：
- ❌ 删除 23 条 AI 填充词（综上所述、值得注意的是等）
- ❌ 删除 18 种 AI 模板段落（首先其次最后等）
- ✅ 引号统一为「」『』
- ✅ 数字汉字化（1987年 → 一九八七年）
- ✅ 字典替换（278 条规则）
- ✅ 语义重写（消除 AI 腔、增强文学质感）

**效果**：
```
删除前：综上所述，他需要做三件事：首先，拿到证据；其次，稳住队友；最后，公开反击。

删除后：他先把证据攥在手里，转身去找队友。人得先稳住，反击才能一刀见血。
```

### Stage 2: 番茄版排版优化

**做什么**：
- ✅ 对话独立成段（前后双换行）
- ✅ 段落最多 80 字、2 句话
- ✅ 删除番茄禁忌词（不知不觉间等）
- ✅ 节奏提速（"他想了想" → "他眼珠一转"）
- ✅ 动作强化（"拿" → "一把抓过"）

**效果**：
```
优化前：他走过去说「你好」然后继续前进。

优化后：
他大步上前。

「你好。」

话音刚落，他闪身逼近。
```

## 常见问题

### Q: 和 /webnovel-chapter 有什么区别？
A: 
- `/webnovel-chapter`：完整写章流程（合同→写作→审查→润色→提交）
- `/webnovel-deslop`：只做润色，不写作，不提交

### Q: 可以用来润色别人写的文本吗？
A: 可以！只要是中文小说文本都可以。

### Q: 会改变剧情吗？
A: 不会。只改表达方式，不改剧情、人物、情节。

### Q: 可以只检测不润色吗？
A: 可以。看完检测报告后，如果不想润色可以停止。

### Q: 润色后的文本在哪里？
A: 
- 如果输入是文件：会询问是否写回文件
- 如果输入是粘贴文本：直接显示在对话中

### Q: 可以调整润色强度吗？
A: 可以。编辑提示词文件：
- `deepseek-universal-polish.yaml` - 调整替换概率
- `tomato-mobile-formatting.yaml` - 调整段落长度等参数

## 技术细节

### 提示词文件位置

```
webnovel-writer/skills/webnovel-write/references/
├── deepseek-universal-polish.yaml      # Stage 1 提示词
└── tomato-mobile-formatting.yaml       # Stage 2 提示词
```

### 执行原理

1. 读取两个提示词文件
2. 将提示词作为 system prompt
3. 将文本作为 user prompt
4. LLM 自动执行润色规则
5. 输出润色后文本

### 与 story-deslop 的区别

| 维度 | story-deslop | webnovel-deslop |
|------|-------------|-----------------|
| 润色引擎 | 手动规则 | DeepSeek 通用 + 番茄版 |
| 提示词 | 内置在 SKILL.md | 独立 YAML 文件 |
| 排版优化 | 无 | 有（番茄版） |
| 字典规则 | 少量 | 278 条 |
| 可配置性 | 低 | 高（可编辑 YAML） |
